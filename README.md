# 岸线 · 27 考研作战手册

面向 2027 考研数学一、英语一与 408 的私人交互式计划网站。默认详细计划覆盖 **2026-07-18 至 2026-11-15**，可在设置中延长，但为避免异常数据拖垮页面，最晚限制为 **2026-12-31**。

在线地址：<https://1850741061.github.io/kaoyan-27-plan/>

## 使用

直接双击 `index.html` 即可使用；也可以在本目录启动静态文件服务器：

```powershell
python -m http.server 4173
```

然后访问 `http://localhost:4173`。

## 主要功能

- 以高数 92%、词汇 35%、数据结构 65%、计组 45% 的当前基线设计 121 天逐日安排，并按“补齐基础、强化成网、真题与报名”三阶段推进；
- 每个学习块都写明范围、目标、可量化完成标准和 3–4 步执行清单；词汇、阅读、翻译、写作、新题型与整卷使用各自的验收方式；
- 周一至周六顺序推进课程，周日固定为约 5.25 小时的周检、唯一补缺和复盘，不推进新课；10 月下旬开始穿插数学一、英语一与 408 的真实 180 分钟整卷；
- 点击“今日执行面”的任一任务，可展开目标、完成标准、分步清单与当日笔记；名称、日期、科目、时间、目标、完成标准、步骤和记录均可编辑；
- 今日执行面、横向日历和一周节律表读取同一份任务数据，任一处完成或编辑后，其他视图立即更新；
- 横向总日历写入每天全部任务：拖动空白处浏览，拖动任务卡可调整日期；
- 一周节律表按所选周实时生成，不是固定模板；可以逐周切换、完成、编辑或从对应时段添加任务；
- 手机端保留任务编辑和删除入口，周表会自动横向定位到当前日期；
- 每日 17:00–19:00 固定运动，学习任务不能覆盖该时段；
- 含报名资料核对、预报名、正式报名和网上确认提醒；相关日期均标注为参考窗口，需以研招网当年公告为准；
- 包含专注计时、分科进度、逐周学习时长与逐周复盘；专注计时按目标时间计算，切到后台或重新开一轮也不会出现负数；
- 支持 JSON 导出与导入；同步冲突、导入和恢复前都会保留可在设置中直接恢复的本机备份；
- 使用 Supabase 邮箱密码登录，在电脑、手机和平板间自动同步；Realtime 负责即时通知，并以聚焦校验和 30 秒轮询兜底；
- 离线或云组件加载失败时仍可进入本机模式，修改会保存在本机，并在下次登录时进入同步流程；
- 云端采用版本号条件更新和串行保存队列：上传过程中产生的新修改会继续上传，不会被前一版误标为已同步；两个设备同时改动时会要求选择保留云端或本机。

逐日课程表单独维护在 `plan-data.js`。任务内容升级时，程序会先自动备份旧计划，保留已经完成、记录或手动编辑过的时间块，再替换其余旧模板任务。

## 首次启用云同步

网站只提供登录，不开放注册。请先进入 Supabase 的 `27plan` 项目：

1. 在 **Authentication → Users** 中添加唯一用户，并设置自己的邮箱和密码；
2. 建议在 **Authentication → Providers → Email** 中关闭新用户注册；
3. 打开网站并登录。首次登录会把当前设备的本机计划上传到云端，之后其他设备登录会读取同一份数据。

## 数据与安全

浏览器 `localStorage` 仍作为离线缓存，Supabase 表 `kaoyan_planner_state` 保存云端副本。该表启用了 RLS，只允许登录用户读写与自己 UID 相同的一行，并已加入 `supabase_realtime` 发布。数据库变更记录位于 `supabase/migrations/`。

前端仅包含 Supabase Project URL 与 publishable key；它们必须配合登录 JWT 和 RLS 才能访问数据。不要把 `service_role` 或 secret key 写入前端。更换浏览器或清除站点数据前，仍建议导出 JSON 备份。

## 本地校验

```powershell
npm run check
npm test
```

`npm test` 会先审计完整 121 天课程表，包括日期覆盖、任务字段、时间冲突、每日负荷、周日恢复规则、阶段边界和报名提醒；随后用两个隔离的虚拟浏览器环境运行同步回归。
